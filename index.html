📐 Chapter 02

The Mathematics Behind the Magic

Every elegant prediction starts with rigorous math. Let's derive everything from first principles.

💰 The Cost Function (MSE)

How do we know if our line is "good"? We measure the error — the difference between predicted values \(\hat{y}_i\) and actual values \(y_i\). We square these errors (to penalize large ones) and average them.

Mean Squared Error (MSE)
🤔 Why square the errors?
  • Makes all errors positive (no cancellation of +/− errors)
  • Penalizes large errors more than small ones (quadratic penalty)
  • Makes the function differentiable everywhere — crucial for optimization
  • Results in a convex function with a single global minimum ✅

📚 Gradient Descent Variants

B

Batch GD

Uses all data points in each update. Stable but slow for large datasets. Guaranteed convergence for convex functions.

S

Stochastic GD

Uses one random data point per update. Noisy but fast. Can escape local minima in non-convex settings.

M

Mini-Batch GD

Uses a small batch (e.g., 32 or 64 points). Best of both worlds — the industry standard.

📋 Chapter 05

The 5 Key Assumptions

Linear regression works beautifully — when its assumptions hold. Violating them leads to unreliable predictions.

📏

1. Linearity

The relationship between X and Y must be linear. Use residual plots to check. If violated → consider polynomial features or transformations.

🎯

2. Independence

Observations must be independent of each other. Common violation: time-series data where consecutive points are correlated (autocorrelation).

📊

3. Homoscedasticity

Residuals should have constant variance across all levels of X. If variance changes (heteroscedasticity) → use Weighted Least Squares or log transform.

🔔

4. Normality of Errors

Residuals should be approximately normally distributed. Important for hypothesis testing & confidence intervals. Check with Q-Q plots.

🔗

5. No Multicollinearity

(For multiple regression) Features should not be highly correlated with each other. Check using VIF (Variance Inflation Factor). VIF > 10 → problem!

📐 Gauss-Markov Theorem

Under assumptions 1–3 (linearity, independence, homoscedasticity), plus the error having zero mean:

Gauss-Markov Conditions

Then the OLS estimators are BLUE — Best Linear Unbiased Estimators. "Best" means minimum variance among all linear unbiased estimators. This is why OLS is so widely used — it's provably optimal under these conditions!

📊 Chapter 06

Evaluation Metrics

How do we know if our model is good? Here are the key metrics every data scientist must know.

MSE

Mean Squared Error

Penalizes large errors quadratically. Most common loss function. Units are squared.

RMSE

Root Mean Squared Error

Same units as Y. More interpretable. Standard metric in Kaggle competitions.

MAE

Mean Absolute Error

Robust to outliers. Doesn't penalize large errors as much. Not differentiable at 0.

R²

R-Squared

0 to 1 scale. 1 = perfect. Can be negative for terrible models. Use Adjusted R² for multiple features.

⚖️ Adjusted R² — The Fairer Metric

R² always increases (or stays the same) when adding more features — even irrelevant ones! Adjusted R² penalizes unnecessary complexity:

🛡️ Chapter 07

Regularization — Taming Overfitting

When your model is too complex, it memorizes noise. Regularization adds a penalty to keep weights small.

L2

Ridge Regression

Adds L2 penalty. Shrinks weights toward zero but never exactly zero. Good when all features are relevant.

L1

Lasso Regression

Adds L1 penalty. Can shrink weights to exactly zero → automatic feature selection! Sparse solutions.

L1+L2

Elastic Net

Combines L1 + L2 penalties. Best of both worlds. Use when features are correlated and you want sparsity.
